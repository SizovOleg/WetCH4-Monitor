## 1. КОНЦЕПТ

### Идея

GEE-инструмент, который показывает пространственно-временную динамику аномалий атмосферного метана над болотами Западной Сибири с привязкой к типологии болотных микроландшафтов и калибровкой по наземным станциям.

### Проблема

Болота ЗС — крупнейший в мире природный источник CH₄. Глобальные продукты (WetCH4, UpCH4) не имеют калибровочных станций в регионе. Регионального инструмента мониторинга не существует. WetCH4 (2025) прямо указывает Western Siberian Lowlands как приоритет для расширения.

### Решение

Связать три слоя данных в одном GEE-приложении:
1. TROPOMI XCH₄ → месячные аномалии ΔCH₄ над болотами
2. Типологическая карта болот (Sentinel-2 + RF) → субпиксельная декомпозиция
3. Наземные данные (Мухрино + публикации) → калибровка и валидация

### Для кого

- Исследователи углеродного цикла бореальных экосистем
- Авторы глобальных моделей CH₄ (WetCH4, GCP) — как региональный benchmark
- Региональные экологические службы (ХМАО, ЯНАО)

### Критерии успеха

1. GEE App работает, карта загружается < 10 сек
2. Карта болот с ≥5 типами, точность ≥ 85%
3. Модель proxy-эмиссии: R² ≥ 0.4 по LOOCV
4. Статья принята в рецензируемый журнал (RSE / Biogeosciences / ESSD)
5. Код открыт, результат воспроизводим

### Чем НЕ является

- Не система оперативного оповещения об утечках
- Не инструмент верификации национальных кадастров
- Не физическая модель переноса метана в атмосфере
- Не альтернатива TROPOMI — надстройка над ним

## 3. ФУНКЦИОНАЛЬНОЕ ОПИСАНИЕ КОМПОНЕНТОВ

### Модуль 1: Wetland Mask (01_wetland_mask.js)

**Роль:** Создаёт карту болот с типологией для всей территории ЗС.

**Что делает:**
- Загружает медианный композит Sentinel-2 за летний сезон (июнь–август)
- Вычисляет спектральные индексы: NDVI, NDWI, MNDWI, NDMI
- Обучает RF-классификатор по обучающей выборке (FeatureCollection с полевыми точками)
- Классифицирует территорию на 9 классов (0 = не болото, 1–8 = типы микроландшафтов)
- Экспортирует результат как ee.Image

**Как работает:** Работает как распознаватель образов — анализирует цвета и влажность поверхности на спутниковых снимках и относит каждый участок к типу болотного микроландшафта. Аналог: BareSand pipeline, адаптированный для болот.

**Ключевые параметры:**
- Каналы: B2, B3, B4, B5, B6, B7, B8, B8A, B11, B12
- Обучающая выборка: ≥ 50 точек на класс
- Деревья в RF: 100

### Модуль 2: TROPOMI Anomaly (02_tropomi_anomaly.js)

**Роль:** Извлекает и обрабатывает данные TROPOMI XCH₄, вычисляет аномалии.

**Что делает:**
- Загружает COPERNICUS/S5P/OFFL/L3_CH4, фильтр qa_value > 0.5
- Агрегирует в месячные медианные композиты
- Вычисляет фон: скользящая медиана по окну ~200 км с маской болот
- Рассчитывает ΔCH₄ = XCH₄ − фон
- Добавляет метаданные: year, month, n_observations

**Как работает:** Берёт спутниковые данные о концентрации метана во всём столбе атмосферы, убирает «общий уровень» (фон) и оставляет только добавку от местных болотных источников.

**Ключевые параметры:**
- qa_value threshold: 0.5
- Окно фона: 200 км (подбирается эмпирически)
- Агрегация: медиана (robust к выбросам)
- Период: май–октябрь (зима исключена)

### Модуль 3: Emission Proxy (03_emission_proxy.js)

**Роль:** Связывает ΔCH₄ с предикторами поверхности через ML-модель.

**Что делает:**
- Собирает предикторы: доля инундации (JRC), T_soil (ERA5-Land), NDVI (MODIS), тип болота (Модуль 1), широта, день года
- Извлекает значения предикторов и ΔCH₄ для точек калибровки
- Обучает ee.Classifier.smileRandomForest
- Применяет модель к территории ЗС
- Вычисляет uncertainty (std по деревьям RF, если доступно)

**Как работает:** Устанавливает эмпирическую связь «если тут такой тип болота, такая температура и такой вегетационный индекс, то аномалия метана обычно такая-то». Потом применяет эту связь ко всей территории.

**Ключевые параметры:**
- Предикторы: 6–8 переменных
- Деревья: 200
- Калибровочные точки: ≥ 30 (из публикаций + Мухрино метео)

### Модуль 4: GEE App (04_app.js)

**Роль:** Интерактивный веб-интерфейс для визуализации результатов.

**Что делает:**
- Рисует карту ΔCH₄ и/или emission proxy
- Предоставляет слайдер по месяцам/годам
- Показывает boxplot ΔCH₄ по типам болот
- Позволяет экспортировать данные для выбранного AOI и периода
- Отображает карту болот как подложку

**Как работает:** Веб-страница, размещённая на серверах Google, которая загружает подготовленные данные и рисует интерактивную карту. Пользователь двигает ползунок времени, кликает на участок — видит характеристики.

## 5. ТЕХНИЧЕСКИЕ ПРИЛОЖЕНИЯ

### 5.1. Алгоритм вычисления фона XCH₄

```
Для каждого пикселя P в месяце M:
1. Определить окно W = 200 км вокруг P
2. Маска: исключить пиксели, где wetland_mask > 0
3. background = median(XCH₄ по W с маской)
4. ΔCH₄(P, M) = XCH₄(P, M) − background
```

Обоснование размера окна: 200 км — компромисс между локальностью (слишком мало пикселей при меньшем окне) и репрезентативностью (слишком далёкий фон при большем). Аналог: Schneising et al. для промышленных источников.
